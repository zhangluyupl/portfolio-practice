## 一、Kev's Dream Team — 多智能体编排典范

**核心定位：** 14+ 个 Agent 组成的工作团队，每个 Agent 专精不同领域

### 团队架构

| Agent | 角色 | 模型 | 职责 |
|-------|------|------|------|
| **Kev** | 编排器/总监 | Opus 4.5 | 接收任务，分解，分配给专家 |
| **Rex** | 软件工程师 | Codex (GPT-5.2) | 执行编码任务 |
| **Hawk** | 安全审计/QA | Opus | PR 审查，安全测试 |
| **Scout** | 研究员 | Gemini Flash | 搜索、文档、情报 |
| **Dash** | 数据分析师 | 数据专项 | SQL、图表、分析 |
| **Dot** | 运维 | IaC 专项 | AWS、Terraform |
| **Pixel** | 设计师 | Gemini Pro | UI/UX、图像生成 |

### 关键架构原则

**① 专业化 > 全能化**
> "你不应该让 CTO 去写 CSS，也不应该让创意总监去做数据库安全审计。"

每个 Agent 只做自己最擅长的事，用最合适的模型。

**② 编排器 > 聊天机器人**
- 聊天：回答问题，一次性
- 编排器：接受任务，拆解，spawn workers，持续到交付

**③ 本地控制 + 云端智能**
- API key 是租来的云端智力
- workspace/记忆/SOUL 必须留在本地
- 执行权在本地，推理在云端

### 技术实现

**Session 工具集：**
```
sessions_spawn   → 派发子任务（并行）
sessions_send    → 跨 Agent 消息
sessions_history → 获取历史
sessions_list    → 查看状态
```

**Docker 沙箱：**
```
sandbox.mode = "all"          → 全部工具跑在沙箱里
sandbox.scope = "agent"       → 按 Agent 隔离
sandbox.workspaceAccess = "rw"
network default = off         → 默认断网
```

**Webhooks → 主动触发：**
```
监控告警 → /hooks/alert → Kev → 派发 Rex/Hawk/Scout
         → 调查 → 修复 → 汇报，全程无人介入
```

**GitHub 为中心的工作流：**
- Issue 定义任务
- `gh` CLI 执行
- PR 为交付物
- Slack 只发完成通知，不发状态更新

### 一个 Ticket 的完整生命周期

```
Adam → WhatsApp → Kev: "需要给 webhook handler 加重试机制"

Kev → 查询 MEMORY.md → 找到代码位置
    → spawn Rex(Codex)：实现重试逻辑
    → spawn Hawk(Opus)：审计 diff
    → Rex → 写测试 → 失败 → 实现 → 测试通过 → 提交 PR
    → Hawk → 检查安全漏洞 → 通过
    → Kev → WhatsApp 回复 Adam: "PR 已就绪，Hawk 已审过"

人类耗时：15 秒
Agent 耗时：3 分钟
```

## 三、对我有启发的案例

### 最值得学习的模式

**① 专业化团队模式**
Kev's Dream Team 证明了：与其用一个通用模型做所有事，不如用专用模型做专项任务。

对我们当前场景的启发：luyu 的 AI 现在是单 Agent，但可以把"研究"和"执行"分开——Scout 模式做调研，Rex 模式做代码。

**② Slack Auto-Support 的自主性**
> "自主修复了生产 bug，没被要求"

这是真正"能做事的 AI"的标准——不是执行命令，是主动发现问题并解决。

**③ 事件驱动的主动响应**
Webhook 触发 → 编排器唤醒 → 派发任务 → 汇报
不需要人类每天盯着，是"always-on"的系统。

## 四、我们的差距分析

| 维度 | 社区标杆 | 我们现状 | 差距 |
|------|---------|---------|------|
| 多 Agent 协作 | Kev's Dream Team 14个Agent | 单 Agent | 很大 |
| 专业化模型选型 | Opus编排+Codex编码+Flash研究 | 单一 MiniMax | 大 |
| 事件驱动 | Webhook → 自动响应 | Cron 轮询 | 中 |
| GitHub 工作流 | PR 即交付物 | 无 | 大 |
| Docker 沙箱 | 生产级隔离 | 无 | 大 |
| 自主发现问题 | Slack Bot 自主修 Bug | 执行命令 | 大 |
