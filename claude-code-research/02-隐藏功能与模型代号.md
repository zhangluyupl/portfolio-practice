# Claude Code 源码研究 — 隐藏功能与模型代号

> 基于 Claude Code v2.1.88 反编译源码分析

## 模型代号体系

Anthropic 使用动物名称作为内部模型代号。

| 代号 | 角色 | 证据 |
|------|------|------|
| **Tengu**（天狗） | 产品/遥测前缀 | 所有 250+ 分析事件和 feature flag 使用 `tengu_*` 前缀 |
| **Capybara**（水豚） | Sonnet 系列模型 v8 | v8 行为问题的 prompt 补丁 |
| **Fennec**（耳廓狐） | Opus 4.6 的前代 | 迁移: `fennec-latest` → `opus` |
| **Numbat**（袋食蚁兽） | 下一代模型 | 注释: "Remove this section when we launch numbat" |

### Capybara v8 的行为问题

1. 停止序列误触发 (~10% 概率)
2. 空 tool_result 导致零输出
3. 过度写注释
4. 高虚假声明率: v8 为 29-30%，而 v4 为 16.7%
5. 验证不足

## Feature Flag 命名约定

所有 feature flag 使用 `tengu_` 前缀 + 随机词对：

| Flag | 用途 |
|------|------|
| `tengu_onyx_plover` | Auto Dream（后台记忆整理）|
| `tengu_coral_fern` | memdir 功能 |
| `tengu_herring_clock` | 团队内存 |
| `tengu_frond_boric` | 分析 kill switch |
| `tengu_amber_quartz_disabled` | 语音模式 kill switch |

## 内外部用户的差异

Anthropic 员工获得显著更好的待遇：更多解释、更严格的虚假声明缓解、更严格的验证、主动指出用户误解。
