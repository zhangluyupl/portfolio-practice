# Claude Code 源码研究 — 遥测与隐私分析

> 基于 Claude Code v2.1.88 反编译源码分析

## 概述

Claude Code 实现了双层分析管道，收集大量环境和使用元数据。虽然没有证据表明存在键盘记录或源代码窃取，但收集范围之广和无法完全退出的事实引发了合理的隐私担忧。

## 数据管道架构

### 第一方日志 (1P)

- **端点**: `https://api.anthropic.com/api/event_logging/batch`
- **协议**: OpenTelemetry + Protocol Buffers
- **批量大小**: 每批最多 200 个事件，每 10 秒刷新一次
- **重试机制**: 二次方退避，最多 8 次尝试，失败事件持久化到磁盘
- **存储位置**: `~/.claude/telemetry/`

### 第三方日志 (Datadog)

- **端点**: `https://http-intake.logs.us5.datadoghq.com/api/v2/logs`
- **范围**: 仅限 64 种预批准事件类型
- **Token**: `pubbbf48e6d78dae54bceaa4acf463299bf`

## 收集了什么

### 环境指纹

每个事件都携带：platform, nodeVersion, 终端类型, 已安装的包管理器, CI/CD 检测, GitHub Actions 元数据, WSL 版本, Linux 发行版, 内核版本, Git 版本, Claude Code 版本和构建时间

### 用户追踪

- 正在使用的模型
- 会话 ID、用户 ID、设备 ID
- 账户 UUID、组织 UUID
- 订阅等级 (max, pro, enterprise, team)
- 仓库远程 URL 哈希 (SHA256 前 16 位)

### 工具输入日志

默认截断到 512 字符，但设置 `OTEL_LOG_TOOL_DETAILS=1` 时**完整工具输入会被记录**。

### 文件扩展名追踪

涉及文件操作的 Bash 命令（rm, mv, cp, touch, mkdir, chmod 等），其文件参数的扩展名会被提取并记录。

## 无法退出的问题

**第一方日志管道无法被关闭**。`isAnalyticsDisabled()` 仅在测试环境、第三方云提供商(Bedrock/Vertex)、全局遥测退出时返回 true。

## GrowthBook A/B 测试

用户在不知情情况下被分配到实验组，系统发送：id, sessionId, deviceID, platform, organizationUUID, subscriptionType

## 关键结论

1. 每个会话收集数百个事件
2. 直接 API 用户无法禁用第一方日志
3. 失败事件保存到磁盘并积极重试
4. 数据发送到 Datadog
5. `OTEL_LOG_TOOL_DETAILS=1` 可启用完整输入记录
6. 仓库 URL 被哈希后发送用于服务端关联
