# Claude Code 源码研究 — 远程控制与紧急开关

> 基于 Claude Code v2.1.88 反编译源码分析

## 远程托管设置

每小时静默轮询 `GET /api/claude_code/settings`，最多 5 次重试。

**"接受否则退出"对话框**：当远程设置包含"危险"变更时，拒绝 = 程序直接退出（exit code 1）。用户只有两个选择：接受，或关掉。

**故障容灾**：远程服务器不可达时使用缓存的旧设置。一旦设置过，就永远无法完全摆脱远程控制。

## Feature Flag 紧急开关（Kill Switches）

| Kill Switch | 用途 |
|-------------|------|
| `tengu_frond_boric` | 分析 sink 禁用 |
| `tengu_amber_quartz_disabled` | 语音模式紧急关闭 |
| bypassPermissionsKillswitch | 绕过权限功能禁用 |
| autoModeCircuitBroken | 阻止重新进入自动模式 |

## 模型覆盖系统

`tengu_ant_model_override` GrowthBook flag 可以远程设置：默认模型、默认 effort level、追加系统提示词、定义自定义模型别名。

## 总结

| 机制 | 范围 | 用户同意 |
|------|------|---------|
| 远程托管设置 | Enterprise/Team | 接受或退出 |
| GrowthBook feature flags | 所有用户 | 无 |
| Kill switches | 所有用户 | 无 |
| 模型覆盖 | 内部 (ant) | 无 |

远程控制在很大程度上没有用户可见性或同意机制。
