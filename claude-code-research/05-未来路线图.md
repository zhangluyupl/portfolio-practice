# Claude Code 源码研究 — 未来路线图

> 基于 Claude Code v2.1.88 反编译源码分析

## 三大方向

### 1. 新模型

- **Numbat（袋食蚁兽）**：下一代模型，代码注释显示发布时将移除 output efficiency 段落
- **Opus 4.7** 和 **Sonnet 4.8** 正在开发中
- 代号演化链：Fennec → Opus 4.6 → Numbat；Capybara → Sonnet v8 → ?

### 2. KAIROS — 自主代理模式（最大未发布特性）

将 Claude Code 从被动助手变为主动自主代理。

**核心特征**：
- 通过 `<tick>` 心跳提示保持活跃
- 根据终端焦点状态调整自主程度（未聚焦=用户离开→大幅自主行动；聚焦=用户在看→更协作）
- 可独立 commit、push 和做决策
- 发送主动通知和状态更新
- 监控 GitHub PR 变更

**关联工具**：SleepTool、SendUserFileTool、PushNotificationTool、SubscribePRTool、BriefTool

### 3. 多模态

- **语音模式**：Push-to-talk 语音输入已完全实现，通过 `VOICE_MODE` feature flag 门控
- **WebBrowserTool**（代号 bagel）：内置浏览器自动化
- **WorkflowTool**：执行预定义工作流脚本
- **TerminalCaptureTool**：终端面板捕获和监控

## 其他未上线功能

- **协调器模式**（COORDINATOR_MODE）：多代理协调系统
- **Buddy 系统**：完整虚拟宠物系统，18 个物种、5 档稀有度、7 种帽子、5 项属性、1% 闪亮概率
- **Dream Task**：后台记忆整固子代理

## 总结

Claude Code 正在从**编程助手**进化为**全天候自主开发代理**。
